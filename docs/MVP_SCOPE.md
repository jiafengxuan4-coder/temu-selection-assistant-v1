# MVP_SCOPE.md

## 产品名称

TEMU 核价选品助手

## MVP 目标

本 MVP 目标是帮助 TEMU 卖家解决选品难和核价通过率低的问题。

用户上传一个已验证有销量表现的爆款产品图片，并输入商品标题、类目、价格等基础信息。系统通过图片识别、爆款元素分析、反比价风险判断，生成多个差异化选品方向，帮助卖家避免直接复制同款，提高核价通过率，并提升找到可售产品的概率。

## 核心痛点

TEMU 卖家目前常见问题：

1. 不知道上什么产品。
2. 直接复制爆款容易被平台反比价系统识别。
3. 只修改标题、图片、描述，核价通过率仍然很低。
4. 一天上架大量产品，但通过核价的产品很少。
5. 通过核价的产品，也不一定具备销售潜力。

因此，本工具的核心价值不是复制爆款，而是拆解爆款元素，并基于爆款元素生成差异化选品方案。

## MVP 必填输入

第一版用户必须提供：

1. 爆款图片
2. 商品标题
3. 商品类目
4. 商品价格

## MVP 选填输入

以下字段为选填：

1. 周销量
2. 月销量
3. 商品评分
4. 评论内容

选填字段的原则：

如果用户提供，系统将用于提高分析准确度。
如果用户不提供，系统仍然可以完成分析，但会在报告中标注数据缺失和置信度降低。

## 图片识别说明

用户上传的图片可能包含：

- 产品本身
- 产品颜色
- 产品款式
- 使用场景
- 组合配件
- 部分销量信息
- 价格信息

但图片中不一定包含：

- 评论内容
- 评分信息
- 完整销量数据

因此，系统不能假设图片里一定有评论和评分。缺失信息必须标记为未知，不得强行编造。

## 销量字段设计

第一版支持：

1. 周销量
2. 月销量

周销量和月销量均为选填。

如果用户未填写销量，系统不应编造销量，只能基于图片、标题、价格、类目进行初步判断，并提示销售潜力判断置信度较低。

## 评论字段设计

评论内容为选填。

如果用户提供评论内容，系统需要分析：

1. 用户喜欢什么
2. 用户吐槽什么
3. 是否存在可改良机会
4. 是否可以生成评论痛点改良款

如果用户未提供评论内容，系统不得强行编造用户痛点。

报告中需要明确显示：

评论数据状态：未提供

并提示：

由于未提供评论内容，本次分析不包含用户真实反馈痛点，仅基于图片、标题、价格、类目和产品结构进行推荐。

## 第一版不做利润计算

第一版 MVP 不包含完整利润计算。

原因：

1. 用户上传的是爆款产品信息，不一定能找到同款货源。
2. 在没有采购价、重量、体积、物流费用的情况下，利润估算不准确。
3. 强行加入利润计算会提高使用门槛，影响 MVP 验证效率。

第一版只做轻量价格段判断。

价格段判断包括：

1. 价格是否偏低
2. 价格是否适合冲量
3. 价格是否适合组合款
4. 价格是否需要更强卖点支撑

完整利润计算放到第二版。

## MVP 核心功能

### 1. 爆款图片识别

系统需要识别：

- 产品类型
- 产品类目
- 产品颜色
- 产品款式
- 是否为单品或组合装
- 是否有配件
- 使用场景
- 图片风格
- 图片是否具备点击吸引力

### 2. 爆款因素分析

系统不强行判断爆款原因，只输出疑似爆款因素。

疑似爆款因素包括：

- 价格因素
- 颜色因素
- 款式因素
- 图片点击率因素
- 综合因素
- 未知因素

每个因素必须包含：

- 因素名称
- 置信度
- 判断理由

### 3. 反比价风险判断

系统需要判断用户如果直接跟款，可能面临的核价风险。

高风险情况包括：

- 直接同款
- 只改标题
- 只改图片
- 只改描述
- 产品结构和组合方式没有变化

降低风险的方向包括：

- 增加配件
- 改成组合套装
- 改变使用场景
- 增加功能
- 改变适用人群
- 改变规格尺寸
- 改变材质
- 根据评论痛点做改良

### 4. 差异化选品推荐

系统需要输出差异化选品方向。

默认模式输出 3 个推荐方向。
高级模式输出 5 个推荐方向。

推荐方向包括：

- 同需求组合款
- 同场景升级款
- 同人群细分款
- 同功能差异款
- 评论痛点改良款

每个推荐方向需要包含：

- 推荐产品方案
- 推荐理由
- 对应爆款元素
- 为什么能降低比价风险
- 为什么仍然有销售机会
- 潜在风险
- 销售潜力分
- 核价通过分
- 综合推荐分
- 推荐等级

## MVP 输出报告

系统最终输出一份结构化选品报告。

报告包含：

1. 爆款基础信息
2. 图片识别结果
3. 疑似爆款因素
4. 爆款类型
5. 数据完整度提示
6. 直接跟款风险
7. 差异化选品方向
8. 每个方向的评分
9. 最终推荐结论
10. 操作建议

## MVP 不包含的功能

第一版不做以下功能：

- 不做自动上架
- 不做自动推广
- 不做浏览器插件
- 不做 TEMU 数据自动采集
- 不做支付系统
- 不做复杂会员系统
- 不做完整数据库后台
- 不做自动生图
- 不做多平台适配
- 不做完整利润计算
- 不做 Y1 / Y2 / 全托管 / 半托管利润模型

## MVP 技术边界

第一版采用：

- Next.js
- TypeScript
- Tailwind CSS
- OpenAI API
- 可选 Supabase Storage
- 可选 Supabase Database

第一版优先保证核心分析流程跑通，不追求复杂 UI。

## MVP 验收标准

MVP 完成后，用户应该能够：

1. 上传一个爆款图片。
2. 输入商品标题、类目、价格。
3. 可选输入周销量、月销量、评分、评论内容。
4. 点击分析按钮。
5. 得到一份结构化选品报告。
6. 看到默认 3 个差异化选品方向。
7. 理解为什么不建议直接复制同款。
8. 理解每个推荐方向为什么可能降低比价风险。
9. 在没有评论、评分、销量的情况下，系统不会强行编造数据。
10. 复制或导出报告。
