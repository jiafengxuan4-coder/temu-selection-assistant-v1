# BUSINESS_RULES.md

## 1. 项目核心定位

本项目是一个面向 TEMU 卖家的核价选品助手。

系统的核心目标不是帮助卖家直接复制爆款，而是帮助卖家拆解爆款元素，并基于爆款元素生成更容易通过核价、同时具备销售机会的差异化选品方向。

第一版 MVP 的核心功能是：

上传一个爆款产品图片，输入商品标题、类目、价格等基础数据，系统输出 3 个差异化选品推荐方向，高级模式输出 5 个推荐方向。

## 2. 核心痛点

TEMU 卖家当前常见痛点：

1. 不知道应该上什么产品。
2. 直接复制爆款容易被平台反比价系统识别。
3. 只修改标题、图片、描述，核价通过率仍然较低。
4. 一天上架大量产品，但通过核价的产品很少。
5. 通过核价的产品，也不一定具备销售潜力。

因此，系统必须同时考虑：

1. 销售潜力
2. 核价通过可能性

第一版不做完整利润计算。

## 3. 输入字段规则

### 3.1 必填字段

第一版 MVP 必填字段：

1. 爆款图片
2. 商品标题
3. 商品类目
4. 商品价格

### 3.2 选填字段

第一版 MVP 选填字段：

1. 周销量
2. 月销量
3. 商品评分
4. 评论内容

如果选填字段缺失，系统仍然可以完成分析，但必须在报告中明确提示数据缺失，并降低相关判断的置信度。

## 4. 图片识别规则

用户上传的图片可能包含：

1. 产品本身
2. 产品颜色
3. 产品款式
4. 产品使用场景
5. 产品组合配件
6. 部分价格或销量信息

但图片中不一定包含：

1. 评论内容
2. 评分信息
3. 完整销量数据

系统不得假设图片中一定存在评论和评分。
如果图片无法识别某些信息，应标记为 unknown，不得强行编造。

## 5. 爆款因素判断规则

系统不能假设每一个爆款都有明确、复杂、可解释的原因。

TEMU 爆款可能由单一因素驱动，也可能由多个因素叠加形成。常见疑似爆款因素包括：

1. 价格因素
2. 颜色因素
3. 款式因素
4. 图片点击率因素
5. 综合因素
6. 未知因素

### 5.1 价格因素

产品价格可能处于用户容易下单的价格区间，或者相对同类产品显得更划算。

但系统不得使用固定价格段判断所有产品的爆款潜力。

不同类目、不同产品、不同客单价区间都可能出现爆款。

价格判断只能作为辅助因素，不能输出绝对结论。

允许输出：

- 价格偏低，可能适合冲量
- 价格适中，适合做组合或升级
- 价格偏高，需要更强卖点支撑
- 当前数据不足，无法判断价格优势

不得输出：

- 这个价格一定能爆
- 这个价格一定不能爆

### 5.2 颜色因素

产品颜色可能影响点击率和购买欲望。

系统应识别：

- 主色
- 辅助色
- 颜色是否醒目
- 颜色是否适合当前类目
- 颜色是否有利于图片点击

### 5.3 款式因素

产品款式、结构、造型、组合方式可能影响销售表现。

系统应识别：

- 是否为常规款
- 是否有特殊结构
- 是否有明显款式差异
- 是否可以做相似款延展
- 是否可以做组合或升级

### 5.4 图片点击率因素

图片点击率是 TEMU 爆款的重要因素之一。

如果图片具备以下特征，可以认为有较高点击潜力：

- 产品主体突出
- 颜色醒目
- 画面干净
- 对比明显
- 一眼能看出产品是什么
- 卖点集中呈现
- 使用场景明确
- 使用人群明确
- 用户只看主图就能理解产品价值

### 5.5 未知因素

真实爆款可能受到平台流量、活动资源、广告投放、短期低价、库存优势等影响。

这些因素如果无法从用户输入中判断，必须归为 unknownFactors，不得强行编造。

## 6. 产品标准化分类规则

系统需要将产品分为：

1. 标品
2. 半标品
3. 非标品

### 6.1 标品

标品指功能明确、规格相对固定、价格透明、同款容易搜索、容易被平台比价的产品。

标品包括但不限于：

- 电水壶
- 充电器
- 手机套
- 手机壳
- 普通手机支架
- 标准电子配件

标品特点：

- 功能明确
- 规格相对固定
- 同款容易找到
- 价格透明
- 1688 上容易搜索到大量类似款
- 核价比价风险较高
- 不适合简单跟款
- 不适合简单组合

标品推荐策略：

- 默认谨慎
- 不建议直接复制同款
- 不建议只改标题、图片或描述
- 除非有工厂定制、功能升级、特殊结构、专属套装或专属包装，否则不优先推荐

### 6.2 半标品

半标品指有明确功能，但仍然可以通过款式、颜色、材质、场景、组合方式、人群细分进行差异化的产品。

半标品包括但不限于：

- 狗牵引绳
- 宠物背带
- 饰品
- 戒指
- 项链
- 帽子
- 部分包包
- 部分家居小物
- 部分宠物用品

半标品特点：

- 有明确功能
- 款式存在变化空间
- 颜色存在变化空间
- 材质存在变化空间
- 使用场景可以被强化
- 可以通过组合或升级降低比价风险

半标品推荐策略：

- 如果输入爆款是单品，优先推荐组合款
- 如果输入爆款已经是组合款，优先推荐升级款
- 可以推荐颜色变化
- 可以推荐材质变化
- 可以推荐多件套
- 可以推荐场景细分
- 可以推荐人群细分

### 6.3 非标品

非标品指款式、颜色、审美、场景、人群影响较大的产品，主要包括服装和服装周边。

非标品包括但不限于：

- 服装
- 女装
- 男装
- 童装
- 服装周边
- 穿搭周边
- 部分风格化配饰

非标品特点：

- 款式变化大
- 颜色影响大
- 审美影响大
- 图片点击率影响大
- 人群定位影响大
- 爆款原因更不确定

非标品推荐策略：

- 重点分析款式
- 重点分析颜色
- 重点分析图片点击率
- 重点分析穿搭场景
- 重点分析目标人群
- 不强行用功能逻辑解释爆款

## 7. 高比价风险判断规则

如果产品满足以下情况，则判定为高比价风险：

1. 产品不是独家款
2. 1688 上一搜就能找到同款
3. 产品价格较低
4. 产品结构简单
5. 产品容易被搬运
6. 产品没有工厂定制
7. 产品没有专属组合
8. 产品没有功能升级
9. 产品没有明显差异化卖点
10. 只是改标题、改图片、改描述

高比价风险典型情况：

- 低成本标品
- 普通小配件
- 普通电子配件
- 普通手机壳
- 普通充电器
- 直接从国内平台搬运的同款产品

系统必须明确提示：

直接跟款风险较高，不建议只改标题、图片或描述后上架。

## 8. 低比价风险判断规则

如果产品具备以下特征，则比价风险较低：

1. 组合套装
2. 工厂专供
3. 工厂定制
4. 专属颜色
5. 专属包装
6. 专属配件
7. 功能升级
8. 场景升级
9. 人群细分
10. 规格差异
11. 材质差异
12. 根据评论痛点改良

低比价风险不代表一定能通过核价，只代表相对直接同款风险更低。

## 9. 推荐方向优先级规则

系统需要先判断用户输入的爆款是：

1. 单品
2. 组合产品
3. 多件套

### 9.1 如果输入爆款是单品

系统优先推荐组合款方向。

原因：

- 单品直接跟款容易被比价
- 单品在 1688 等平台更容易找到同款
- 单品价格透明，核价风险较高
- 通过增加配件和组合，可以形成新的 SKU
- 组合款更容易表达使用场景和卖点

默认推荐顺序：

1. 组合款方向
2. 功能升级方向
3. 场景 / 人群细分方向

### 9.2 如果输入爆款已经是组合产品

系统优先推荐升级款方向。

原因：

- 原爆款已经通过组合形成差异
- 继续做相似组合，仍可能存在比价风险
- 升级功能、场景、人群或材质，更容易形成新的差异化

默认推荐顺序：

1. 功能升级方向
2. 场景 / 人群细分方向
3. 图片表达迁移方向

### 9.3 如果输入爆款是标品

系统默认谨慎。

默认推荐顺序：

1. 功能升级方向
2. 定制 / 特殊结构方向
3. 场景化包装方向

如果无法找到明确升级或定制机会，应输出“不建议直接跟款”。

### 9.4 如果输入爆款是非标品

系统重点分析：

1. 款式
2. 颜色
3. 图片点击率
4. 穿搭场景
5. 目标人群

默认推荐顺序：

1. 款式延展方向
2. 颜色 / 风格迁移方向
3. 人群 / 场景细分方向

## 10. 组合推荐规则

系统推荐组合款时，需要优先考虑以下组合方式：

### 10.1 同场景组合

多个产品服务于同一个使用场景。

例如：

- 狗牵引绳 + 狗背带 + 拾便袋
- 帽子 + 围巾
- 帽子 + 包包
- 露营灯 + 收纳袋

### 10.2 同人群组合

多个产品服务于同一个目标人群。

例如：

- 养狗人群
- 户外露营人群
- 宝妈人群
- 办公室人群
- 旅行人群

### 10.3 同功能延伸组合

主产品之外，增加辅助功能配件。

例如：

- 牵引绳 + 拾便袋
- 水杯 + 清洁刷
- 收纳盒 + 标签贴
- 首饰 + 收纳袋

### 10.4 多件套组合

同类产品通过数量、颜色、款式做组合。

例如：

- 戒指 2 件套 / 3 件套
- 发夹多色套装
- 袜子多双组合
- 收纳袋多尺寸组合

## 11. 升级款推荐规则

适合升级的产品通常具备以下特征：

1. 原爆款有明确使用场景
2. 用户有潜在痛点
3. 产品结构允许增加功能
4. 产品可以通过材质、颜色、尺寸、配件、功能做差异化
5. 升级后用户能够一眼看出价值

常见升级方向：

1. 材质升级
2. 颜色升级
3. 功能升级
4. 尺寸升级
5. 场景升级
6. 人群细分
7. 配件增加
8. 包装升级

例如：

- 普通狗牵引绳 → 反光夜光牵引绳
- 普通狗牵引绳 → 防爆冲缓冲牵引绳
- 普通帽子 → 防晒透气户外帽
- 普通戒指 → 多件套可叠戴戒指

## 12. 不建议推荐的产品

系统不优先推荐以下产品：

### 12.1 太容易复制的低价标品

特征：

- 1688 大量同款
- 单价低
- 结构简单
- 无功能差异
- 无组合空间
- 无品牌壁垒
- 无供应链优势

### 12.2 没有明显使用场景的产品

如果产品无法清晰表达使用场景，用户不容易理解为什么要买。

### 12.3 只能靠低价竞争的产品

如果一个产品只能通过价格竞争，而无法通过款式、组合、功能、图片、场景提高价值，不建议优先做。

### 12.4 难以做差异化的强标品

如果系统无法找到组合、升级、场景、人群、定制方向，应明确提示不建议直接跟款。

## 13. 评论数据规则

评论内容是选填字段。

如果用户提供评论内容，系统需要分析：

1. 用户喜欢什么
2. 用户吐槽什么
3. 是否存在可改良机会
4. 是否可以生成评论痛点改良款

如果用户未提供评论内容，系统不得强行编造用户痛点。

报告中需要明确显示：

评论数据状态：未提供

并提示：

由于未提供评论内容，本次分析不包含用户真实反馈痛点，仅基于图片、标题、价格、类目和产品结构进行推荐。

## 14. 销量数据规则

第一版支持：

1. 周销量
2. 月销量

周销量和月销量均为选填。

如果用户填写周销 / 月销，则参与销售潜力判断。

如果用户未填写销量，系统不应编造销量，只能基于图片、标题、价格、类目进行初步判断，并提示销售潜力判断置信度较低。

## 15. 第一版不做利润计算

第一版 MVP 不包含完整利润计算。

原因：

1. 用户上传的是爆款产品信息，不一定能找到同款货源。
2. 在没有采购价、重量、体积、物流费用的情况下，利润估算不准确。
3. 强行加入利润计算会提高使用门槛，影响 MVP 验证效率。

第一版只做轻量价格段判断。

完整利润计算放到第二版。

## 16. 评分规则

第一版每个推荐方向输出三个分数：

1. 销售潜力分
2. 核价通过分
3. 综合推荐分

### 16.1 销售潜力分

销售潜力分用于判断该推荐方向是否有卖起来的机会。

参考因素：

1. 原爆款销量表现
2. 原爆款价格是否有吸引力
3. 图片点击潜力
4. 使用场景是否明确
5. 目标人群是否明确
6. 产品需求是否稳定
7. 是否具备可迁移的爆款元素

如果缺少销量、评分、评论数据，销售潜力分可以正常输出，但必须降低置信度。

### 16.2 核价通过分

核价通过分用于判断该推荐方向相对于直接同款，是否更容易降低比价风险。

参考因素：

1. 是否避免直接同款
2. 是否增加组合
3. 是否有功能升级
4. 是否有人群细分
5. 是否有场景变化
6. 是否有颜色 / 材质 / 规格差异
7. 是否降低 1688 同款比价风险
8. 是否形成新的 SKU 方案

### 16.3 综合推荐分

综合推荐分计算方式：

综合推荐分 = 销售潜力分 × 40% + 核价通过分 × 60%

原因：

本工具第一版的核心痛点是帮助 TEMU 卖家提高核价通过率，因此核价通过分权重高于销售潜力分。

### 16.4 推荐等级

根据综合推荐分输出推荐等级：

- 80-100：推荐优先测试
- 60-79：可以小批量测试
- 40-59：谨慎，不建议优先做
- 0-39：不建议做

## 17. 输出数量规则

第一版 MVP 输出：

- 默认模式：3 个推荐方向
- 高级模式：5 个推荐方向

系统不应为了凑数量输出低质量推荐。

如果有效推荐不足，应明确说明：

当前数据下可推荐方向有限，建议补充销量、评论或更多图片后重新分析。

## 18. 不得强行编造规则

系统必须遵守以下原则：

1. 不得编造销量
2. 不得编造评分
3. 不得编造评论
4. 不得编造用户痛点
5. 不得把不确定因素说成确定结论
6. 不得保证某个产品一定能通过核价
7. 不得保证某个产品一定能爆

系统应使用以下表达：

- 疑似
- 可能
- 倾向于
- 置信度较高
- 置信度较低
- 当前数据不足

系统不得使用以下表达：

- 一定
- 必然
- 保证
- 绝对
- 百分百
